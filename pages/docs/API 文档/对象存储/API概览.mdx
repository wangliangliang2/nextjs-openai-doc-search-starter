# **Bucket API** 

| 接口   | 说明 |
| -----|-------------------|
|[mkbucketv3](/kodo/api/1382/mkbucketv3)        | 创建新的存储空间|
|[drop](/kodo/api/1601/drop-bucket)        | 删除指定存储空间|
|[domains](/kodo/api/3949/get-the-bucket-space-domain)        | 获取一个空间绑定的所有域名列表|

|[deleteBucketTagging](/kodo/api/6316/delete-bucket-tagging)        | 删除空间标签|

<a id="object-operation"></a>
# **Object API** 

| 接口   | 说明 |
| -----|-------------------|
|[upload](/kodo/api/1312/upload)        | 用于在一次 HTTP 会话中上传单一的一个对象|
|[mkblk](/kodo/api/1286/mkblk)  |  为后续分片上传创建一个新的块，同时上传第一片数据|
|[bput](/kodo/api/1251/bput)    |上传指定块的一片数据，具体数据量可根据现场环境调整，同一块的每片数据必须串行上传|
|[mkfile](/kodo/api/1287/mkfile)  | 将上传好的所有数据块按指定顺序合并成一个对象资源|
|[initiateMultipartUpload](/kodo/api/6365/initialize-multipartupload)        | 通知服务端开启分块上传任务,得到全局唯一任务UploadId|
|[uploadPart](/kodo/api/6366/upload-part)        | 分块上传数据，需指定的任务UploadId|
|[completeMultipartUpload](/kodo/api/6368/complete-multipart-upload)        | 完成整个对象的分块上传，需指定的任务UploadId|
|[abortMultipartUpload](/kodo/api/6367/abort-multipart-upload) | 中止分块上传任务，并且删除已经上传的块，需指定的任务UploadId|
|[listParts](/kodo/api/6858/listparts)|列举指定 UploadId 所属的所有已经上传成功Part|
|[list](/kodo/api/1284/list) | 用于列举指定空间里的所有对象条目|
|[stat](/kodo/api/1308/stat)| 仅获取对象的 Metadata 信息，不返回对象内容|
|[chgm](/kodo/api/1252/chgm) |修改对象的 MIME 类型信息|
|[move](/kodo/api/1288/move) | 将源空间的指定对象移动到目标空间，或在同一空间内对对象重命名|
|[copy](/kodo/api/1254/copy) | 将指定对象复制为新命名对象|
|[delete](/kodo/api/1257/delete) | 删除指定对象|
|[batch](/kodo/api/1250/batch) | 指在单一请求中执行多次获取元信息、移动、复制、删除操作，极大提高对象管理效率|
|[chstatus](/kodo/api/4173/modify-the-file-status)|修改对象的存储状态，即切换启用、禁用状态|
|[chtype](/kodo/api/3710/chtype) | 修改对象的存储类型信息，即低频存储、标准存储和归档存储的互相转换|
|[restoreAr](/kodo/api/6380/restore-archive)|解冻归档存储类型的对象|
|[deleteAfterDays](/kodo/api/1732/update-file-lifecycle) | 设置一个对象多少天后过期删除|
| [lifecycle](/kodo/api/8062/modify-object-life-cycle)|修改已上传对象的生命周期，即修改对象转低频时间、转归档时间、过期删除时间|
|[prefetch](/kodo/api/1293/prefetch) | 对于设置了镜像存储的空间，从镜像源站抓取指定名称的对象并存储到该空间中|
|[sisyphus/fetch](/kodo/api/4097/asynch-fetch) | 从指定 URL 抓取对象，并将该对象存储到指定空间中。每次只抓取一个对象，抓取时可以指定保存空间名和最终对象名|
