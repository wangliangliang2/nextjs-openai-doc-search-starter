<a id="abortMultipartUpload-description"></a>
# **描述** 

该接口根据 UploadId 终止 Multipart Upload 。


<br>

<a id="abortMultipartUpload-requests"></a>
# **请求**
<a id="abortMultipartUpload-requests-syntax"></a>

### **请求语法**

```
DELETE /buckets/<BucketName>/objects/<EncodedObjectName>/uploads/<UploadId> HTTP/1.1
Host: <UpHost>
Authorization: UpToken <UploadToken>
```
使用说明：
* 当调用该 API 终止 Multipart Upload 后，无法再使用这个 UploadId 做任何操作，已经上传的 Part 数据也会被删除。


<br>

<a id="abortMultipartUpload-request-parameters"></a>
### **请求参数**

| 参数名称 | 必填 | 说明 |
| --- | --- | --- |
| BucketName | 是 | 空间名称|
| EncodedObjectName | 否 | 资源名，EncodedObjectName 需要经过 Base64 编码。具体可以参照：[URL 安全的 Base64 编码](/kodo/manual/1231/appendix#urlsafe-base64) |
| UploadId | 是 | 在服务端申请的 Multipart Upload 任务 id |

<br>

<a id="abortMultipartUpload-request-headers"></a>
### **请求头**
|头部名称       | 必填 | 说明|
|------------- | --- | -------------------------------- | 
|Host           | 是   | 上传域名。<span id="Endpoint">七牛对象存储支持的区域和对应区域上传域名列表见 [存储区域列表](/kodo/1671/region-endpoint-fq)</span> |
|Authorization  | 是   | 该参数应严格按照[上传凭证](/kodo/manual/1208/upload-token)格式进行填充，否则会返回 401 错误码。一个合法的 Authorization 值应类似于 Authorization: UpToken QNJi_bYJlmO5LeY08FfoNj9w_r... |

<br>

<a id="abortMultipartUpload-request-elements"></a>
### **请求内容**
