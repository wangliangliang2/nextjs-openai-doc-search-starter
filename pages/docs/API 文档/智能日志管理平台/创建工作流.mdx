**请求语法**

```
POST /v2/workflows/<Workflow>
Content-Type: application/json
Authorization: Pandora <auth>
{
    "region": <Region>,
    "workflow": <Workflow>,
    "comment": <Comment>
}
```

**请求内容**

|参数|类型|必填|说明|
|:---|:---|:---:|:---|
|Workflow|string|是|工作流名称</br>命名规则: `^[a-zA-Z_][a-zA-Z0-9_]{0,127}$`</br>1-128个字符，支持小写字母、数字、下划线</br>必须以大小写字母或下划线开头，且该工作流必须提前创建|
|region|string|是|计算与存储所使用的物理资源所在区域</br>|
|comment|string|否|当前工作流的备注|


> 注意:

1. `region`参数是为了降低用户传输数据的成本，请尽量选择离自己数据源较近的区域。

```
-d '{
    "region": "nb", 
	 "workflow": "test",
    "comment": "workflow_for_test"
}'
```
