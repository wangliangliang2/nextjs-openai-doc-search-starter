# 请求路径
> `GET` /v1/ess/group/:id/rule

# 请求参数

| 名称 | 类型 | 位置 | 是否必须 | <center>描述 |
| ----- | ----- | ----- | ----- | ----- |
| region_id | string | query | 是 | 地域 ID |
| id | string | path | 是 | 伸缩组 ID |
| scaling_rule_id | string | query | 否 | 待查询伸缩规则的ID，最多可以输入10个，用逗号','分隔。 |
| scaling_rule_name | string | query | 否 | 待查询伸缩规则的名称，最多可以输入10个，用逗号','分隔。 |
| scaling_rule_ari | string | query | 否 | 待查询伸缩规则的唯一标识符，最多可以输入10个，用逗号','分隔。 |
| scaling_rule_type | string | query | 否 | 伸缩规则的类型。取值范围：SimpleScalingRule：简单规则; TargetTrackingScalingRule：目标追踪规则; StepScalingRule：步进规则; PredictiveScalingRule：预测规则。 |
| page | int | query | 否 | 页码 |
| size | int | query | 否 | 分页大小 |

# 返回值

| 名称 | 类型 | 是否必须 | <center>描述 |
| ----- | ----- | ----- | ----- | ----- |
| data | []ScalingRule | 是 | 伸缩组规则 |
| request_id | string | 是 | 请求记录 ID |
| page | int | 是 | 页码 |
| size | int | 是 | 分页大小 |
| total | int | 是 | 查询到的总数 |


### ScalingRule 伸缩组规则详情
| 名称 | 类型 | 是否必须 | <center>描述 |
| ----- | ----- | ----- | ----- |
| scaling_rule_id | string | 是 | 伸缩组规则 ID |
| scaling_group_id | string | 是 | 伸缩组 ID |
| scaling_rule_name | string | 是 | 缩规则的名称 |
| scaling_rule_ari | string | 是 | 伸缩规则的唯一标识符 |
| scaling_rule_type | string | 是 | 伸缩规则的类型。 |
| adjustment_type | string | 是 | 伸缩规则的调整方式。可选值： QuantityChangeInCapacity：增加或减少指定数量的 ECS 实例; PercentChangeInCapacity：增加或减少指定比例的 ECS 实例; TotalCapacity： 将当前伸缩组的 ECS 实例数量调整到指定数量。|
| adjustment_value | int | 是 | 伸缩规则的调整值。任何情况下，单次调整的 ECS 实例台数都不能超过 1000。不同调整方式对应的取值范围： QuantityChangeInCapacity：-500~500 PercentChangeInCapacity：-100~10000 TotalCapacity：0~1000 |
| cooldown | int | 是 | 伸缩规则的冷却时间。 取值范围：0~86400，单位：秒。 |
