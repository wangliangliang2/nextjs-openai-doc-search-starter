七牛云 QCS 实例的生命周期是指 QCS 实例从启动到销毁所经历的状态。通过使用七牛云云主机从启动到销毁期间对实例进行合理的管理，可确保运行于云主机上的应用程序高效经济地提供服务。

下图显示实例状态之间的转换。有关包年包月类型实例和按量计费类型实例的更多内容，可以参考 [计费模式说明](/fec/manual/pricing-model)。

QCS 实例有以下状态：

- 创建中 
- 运行中 
- 重启中
- 关机中 
- 已关机 
- 重置中
- 销毁中 
- 已销毁 

## 实例启动

执行启动实例操作后实例将进入启动中（pending）状态。启动时指定的 [实例类型](/fec/manual/instance-type-overview) 将决定实例的硬件配置，系统将使用在启动时指定的镜像来启动实例，且可能要过一小段时间才能连接到实例，当实例准备就绪后将进入运行中（running）状态。

实例会获得一个私有 IP 地址，相同地域相同网络环境内（VPC）的其他云主机可用其与之通信；若选择使用了 [弹性公网 IP](/fec/manual/elastic-public-ip) 还会获得一个公网 IP 地址，可用于从 Internet 与该实例通信。用户可以连接到正在运行的实例，然后像使用普通的计算机一样使用它。

更多信息请参考 [购买并启动实例](/fec/manual/buy-and-start-the-instance)、[登录 Windows 实例](/fec/manual/1877/login-windows-instance) 和 [登录 Linux 实例](/fec/manual/1865/login-linux-instance)。

## 实例重启

我们建议用户任意选择七牛云控制台来重新启动实例，而非在实例中运行操作系统重启命令。执行重启实例操作后实例将进入重启中状态。

重启实例相当于重启计算机，重启后实例仍保留其公有 IP 地址、私有 IP 地址以及其硬盘上的所有数据。完成重启通常需要花费几十秒至几分钟的时间，该时间具体取决于实例配置。

更多信息请参阅 [重启实例](/fec/manual/restart-the-instance-2)。

## 实例关机

当用户某一时刻需要停止服务时，可以对实例执行关机操作。用户可以任意选择使用七牛云控制台来关机实例。

关机后的云主机实例仍会在控制台中显示。<font color="red">不管是否关机，按量计费类型实例会持续收费直至用户销毁它。</font>包年包月类型实例到期后的一段时间内也会呈现关机状态并将在过期七天后回收。需要注意的是，用户 **_无法_** 连接已关机的实例。

关机实例相当于关闭计算机，关机是某些操作的前提（如调整配置、重置密码等）。关机操作本身并不会改变云主机的公有 IP 地址、私有 IP 地址及其硬盘上的所有数据，但后续操作（调整硬件配置等）可能导致 IP 的改变，请用户参考相应操作说明。

更多信息请参阅 [关机实例](/fec/manual/shut-down-the-instance-2)。

## 实例销毁

用户不再需要云主机实例时，可以终止该实例。其中：

- 不再需要包年包月类型实例时，实例到期七天后自动销毁。
- 不再需要按量计费类型实例时，需要用户手动销毁该实例，可通过七牛云控制台实现。

实例销毁时实例的系统盘及购买时指定的数据盘会随之销毁，但挂载在其上的弹性云盘不受影响。

更多信息请参阅[销毁实例](/fec/manual/destroy-the-volume-pricing-instance)。
