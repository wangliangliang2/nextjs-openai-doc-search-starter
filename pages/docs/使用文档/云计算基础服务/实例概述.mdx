## 什么是 QCS 实例？

用户可以使用同一个镜像启动不同类型的实例。[实例类型](/fec/manual/instance-type-overview) 决定了用于实例的主机硬件配置。每一个实例类型提供不同的计算和存储能力，用户可以基于需要提供的服务规模而选择实例计算能力、存储空间和网络访问方式。有关每种实例类型的更多信息，请参阅 [QCS 实例配置](/fec/manual/qcs-instance-configuration)。实例启动后用户即可像使用传统计算机一样使用它，用户对启动的实例有完全的控制权。

## 什么是镜像？

镜像是一种云主机软件配置（操作系统、预安装程序等）的模板。七牛云要求用户通过镜像启动实例。镜像可以启动多个实例，供用户反复多次使用。

七牛云提供公共镜像和自定义镜像（私有）。前者包含了标准的操作系统，为用户提供了标准运行环境。

自定义镜像能让用户快速启动满足特定需求的新实例。例如，如果用户的应用程序是网站或 Web 服务，则您可以创建包含 Web 服务器、相关静态内容和动态页面代码的自定义镜像。因此，当从这个镜像启动实例时这些 Web 应用已经部署完成可以提供服务。

## 实例的存储

实例像普通计算机一样，有系统运行盘（如 Windows 系统下的 `C 盘`一样）和数据盘（如 Windows 系统下的其他 `D 盘`、`E 盘`一样）之分，在实例中分别称为系统盘和数据盘。

系统盘中包含用于启动实例的镜像的完全副本（因此启动时必须选择大于使用镜像的系统盘大小）以及实例运行环境。数据盘保存用户数据，支持自由地扩容、挂载和卸载。

系统盘和数据盘都可以使用七牛云提供的不同存储类型。有关更多信息，请参阅 [存储](/fec/manual/summary-of-storage)。

## 实例的安全

- 通过使用防火墙（安全组）允许受信任的地址访问实例来限制访问。例如，您可以通过限制端口 80 的入站流量来限制外部对本实例的 HTTP 访问。有关更多信息，请参阅 [安全组](/fec/manual/the-security-group)。
- 创建不同的安全组规则应用于不同安全级别的实例组上，确保运行重要业务的实例无法轻易从外部触达。
- 尽量使用 [SSH 密钥](/fec/manual/ssh-key)方式登录用户的 Linux 类型实例，使用 [密码登录](/fec/manual/the-login-password) 的实例需要不定期修改密码。
