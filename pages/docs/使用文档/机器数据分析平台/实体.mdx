## **概览**
实体是一个监控的主体。每个实体都具有特定属性，并与对其唯一性标识 的其他 IT 流程之间存在特定关系。实体是客观存在并可相互区别的事物。
实体通常是主机，但也可以是网络设备、应用程序、用户等。实体可以是以下组件中的任一项:
- 物理或虚拟主机
- 网络设备(交换机、路由器)
- 用户 (AD/LDAP)
- 存储系统、卷
- 操作系统进程


在PISA中，新建实体有2种方式
- 定义单个实体
- [导入实体](/express/manual/9665/import-service-entities)

## 定义单个实体
进入Pandora智能服务分析App，单击实体。选择新建实体，配置以下配置项以定义一个新的实体

| 配置项 | 是否必填 | 配置项解释 |
|-|-|-|
|  名称       | 是      | 实体名称，全局唯一 | 
| 描述   | 否       | 实体描述      | 
| 匹配条件    | 是       | 在数据中识别实体的条件，由K-V组成，可添加多个K-V，至少添加一个。当数据中满足的存在其中填写的一个K-V，则表示匹配上该实体 |   
| 属性    | 是       | 实体的附加属性信息，由K-V组成，可添加多个K-V。                       | 
| 标签 | 否      | 可以给实体添加标签，可选择标签或者新建一个标签                                 |                             

## **导入实体**
[导入实体](/express/manual/9665/import-service-entities)需要经历3步：选择数据源、指定导入信息、设置定时导入。支持用户从搜索、数据集、CSV中导入实体，无需固定的文件格式，主要思想是将常规的表格结构转化为实体结构。


## **实体管理**
在实体管理页面，可以选择实体进行编辑（不可编辑名称）、导出（支持导出单个、多个、全部）、删除（支持删除单个、多个）、查看详情。同时支持按照名称、标签进行实体模糊搜索

## **查看实体详情**
点击某个实体操作栏的实体详情按钮，可以查看当前实体的详细信息，包括所有的配置信息：名称、描述、匹配条件、属性、标签
