# 新功能

- 主页大改版
  - 产品界面展现重点从数据处理转换为各APP应用，标志着Pandora在产品定位上的方向转移；
  - 未来Pandora在保持稳定底层数据处理服务之上，支持多方向业务应用，服务好各层次用户（工程师、合作伙伴及最终客户），形成多层次的服务体系：
    - 上层方便最终客户直接触达业务应用
    - 中层支持合作伙伴二次开发多业务场景app
    - 底层保障大批量多种类响应快的数据服务
- 权限改版
  - 新增仓库权限，控制不同用户对不同数据仓库的访问权限
  - 新增应用权限，控制不同用户对不同应用的访问/编辑权限
  - 新增知识对象权限作用域：
    - 知识对象创建时需要选择目标APP，即知识对象应用上下文
    - 知识对象授权时需要考虑应用的作用域，创建时默认为私有，需要共享时考虑在APP内生效还是全局（所有应用）生效
    - 知识对象可能存在同名，使用有优先级
- 新增字段查找（lookup)：
  - 满足部分场景中通过字段匹配查找外部数据源（查找表：文件/外部脚本如数据库/kv存储），在搜索结果中增加相关信息以丰富事件数据的需求。
- 新增低代码组件
  - 新增低代码组件：图片(img), 网格格布局(GridLayout)，滑动条(slider)，方便用户在进行APP视图研发时更快捷创建这些组件。
 
# 功能优化

- 任务管理
  - SPL任务记录：新增应用和任务类型两个维度筛选，方便用户统一查看及批量处理，也方便用户缩小处理范围&了解问题根源。
  - 调度任务记录：XIHE调度平台运行任务记录的运维操作管理，目前仅管理PISA定时调度任务。
- 低代码功能优化
  - 迁移低代码组件：dashboard，chart，微应用，view-condition，event-search，logdb，event，visualization-search等，提升这些组件的创建及编辑的便捷性；
  - 功能改造：组件之间通信能力（trigger）增强
  - xml增加版本号
- 告警服务
  - 告警规则列表新增状态筛选
  - 告警抑制功能联动告警级别，只显示当前告警定义的等级
  

# 性能增强

- 向量化引擎灰度发布，大幅提升分析效率：
   - 新增支持算子  
     - sort：支持单字段排序、多字段排序、非排序字段延迟加载，整体性能大幅提升；
     - bin：对比老引擎，时序数据分桶性能提升10倍+，数值数据分桶性能提升20倍+；
     - convert：对比老引擎，字段值转换为数值性能提升约2倍。
  - 优化：谓词下推优化
  - bugfix: 修复 eval 中 boolean dictionary 的构建

- 单值图前后缀字体显示异常问题修复；
- 告警指标中的告警对象不显示的问题修复；
- 告警规则边上的刷新按钮按下后告警规则详情页面没刷新的问题修复；
- 搜索方式给角色分配新的仓库权限会覆盖掉旧的权限的问题修复；
- 新主页视图无法选择已添加相关应用视图的问题修复。
