# **用户**
用户对应某一个操作实体，如软件开发人员。通过创建新的用户并授权，用户便具有相关资源的操作权限。

## **创建用户**
1.在权限管理模块，单击**用户管理>新建用户**。
2.按照页面提示，输入用户名、邮箱、登录密码、所属用户组等信息。
![](https://pandora-kibana.qiniu.com/authorization/create_user.png)
## **管理用户**
用户如果有`编辑用户组内数据权限`，即可对用户的名称、手机号、登录密码、所属用户组信息进行修改。
若子账号想将用户分配至多个用户组，需同时具有被添加用户组的读取、编辑权限。
> 注意：用户一经创建，邮箱信息不可更改。
## **删除用户**

1.在权限系统模块的用户管理页面，找到要删除的用户。
2.点击删除。
注意：删除用户需要谨慎操作。如果有用户正在使用此子账号操作数据，直接删除可能会导致客户的业务故障。

## **用户授权** 

对用户授权需把用户添加至用户组，通过将角色分配给用户组实现对用户的授权。

# **用户组**

权限系统通过将角色分配给用户组实现对用户的授权。

以用户组为基本单位授权的好处如下：

1.当用户职责发生变化时，只需将其移动只相应职责的用户组里即可，无需重复对单个用户分配角色的操作。

2.当用户组的权限发生变化时，只需修改用户组的角色，即可应用到组内全部用户。

## **创建用户组**

1.在权限管理模块，单击**用户组>新建用户**。
2.输入用户组名称、描述，选择用户组的用户和角色，点击确定。

![](https://pandora-kibana.qiniu.com/authorization/create_ug.png)

## **管理用户组**

用户如果有`编辑用户组权限`，即可对用户组名称、描述、组内成员、角色进行修改。

需注意的是，

1.若子账号想往用户组添加用户，需同时具有被添加用户的读权限。
2.若子账号想往用户组添加、删除角色，需具有相应角色的编辑权限。

## **删除用户组**

删除用户组会将用户从组内移出，并解除组内用户的授权关系。

关于如何为用户组授权，请参考[角色授权>用户组授权](/insight/manual/5783/role-management)。
