您可以通过如下步骤完成对子账号的授权：

<a href="#1">1.创建资源组，将资源添加进指定资源组。</a>
<a href="#2">2.创建角色，对角色进行资源的授权。</a>
<a href="#3">3.创建用户（子账号）。</a>
<a href="#4">4.创建用户组，添加组内用户，并给用户组分配角色。</a>
<a href="#5">5.子账号登陆智能日志平台操作资源。</a>

-----

<a name="1"></a>

### **第一步，将目标授权资源添加进资源组**

登录智能日志平台，单击**权限管理 > 资源组 > 添加资源组**。将您想要授权的资源添加到资源组。

![](https://pandora-kibana.qiniu.com/authorization/new_rg.png)

<a name="2"></a>

### **第二步，创建角色并设置角色的权限**

单击**权限管理 > 角色 > 创建角色**，创建角色并设置角色的权限。

* 对资源组授权：对角色授予创建资源组、创建资源、资源组的组管理权限和组内数据管理权限，在相应的权限前面打勾即可。
 这里我们对项目A管理员授予项目A资源组的查看编辑权限，以及项目A资源组内数据的查看、写入、编辑、删除权限。

![](https://pandora-kibana.qiniu.com/authorization/new_role.png)

* 对用户组授权：对角色授予创建用户组、创建用户、用户组的组管理权限和组内数据管理权限，在相应的权限前面打勾即可。
  这里我们对项目A管理员授予项目A用户组的查看、编辑权限和组内成员的查看、编辑权限。

![](https://pandora-kibana.qiniu.com/authorization/ug_permission.png)

* 对功能模块授权：设置角色允许访问的功能模块。
  这里我们对项目A管理员授予数据上传模块、日志仓库模块、实时仓库模块、仪表盘模块、工作流模块、权限管理模块的访问权限。

![](https://pandora-kibana.qiniu.com/authorization/url_permission.png)

* 角色管理：对角色授予角色的创建、查看、编辑、删除权限。
  这里不做设置。

关于权限详情请阅读[角色权限](/insight/manual/5783/role-management)。

<a name="3"></a>

### **第三步，创建子账号，即您的授权对象**

登录智能日志平台，单击**权限管理 > 用户管理 > 新建用户**。按照页面提示，填写用户名、邮箱、登录密码、用户组等信息。

>注意：权限系统仅对用户组分配角色，所以用户实体需归属于某个用户组才能拥有权限。

![](https://pandora-kibana.qiniu.com/create_sub.png)

<a name="4"></a>

### **第四步，创建用户组**

登录智能日志平台，单击**权限管理 > 用户组 > 新建用户组**。选择第三步创建的用户和第二步创建的角色添加进用户组。

![](https://pandora-kibana.qiniu.com/authorization/ug_create.png)

> 注意：创建用户组需给用户组分配一个角色，以便用户组中的用户拥有角色中设定的权限。

<a name="5"></a>

### **第五步，子账号登录智能日志平台操作资源**

根据主账号授予的权限，子账号登录智能日志平台可以看到如下功能模块。
且具有资源组内数据的查看、写入、编辑、删除权限。

![](https://pandora-kibana.qiniu.com/authorization/sub_r.png)
具有用户组的查看、编辑权限。
![](https://pandora-kibana.qiniu.com/authorization/sub_ug.png)

现在，您可以自行尝试创建一个子账号并对其授权，体验权限系统的功能。
