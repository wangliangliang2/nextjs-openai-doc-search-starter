智能日志平台使用`按实际用量收费`的计费方式，每个月有一定的免费额度。本文档介绍实时仓库、日志仓库、数据推送、数据计算、数据投递等产品的计费方式及示例。首先，您需要了解：

* `每月`提供账单，按当月的资源使用量进行收费。

* 免费额度的周期为月，当月结束后会清零。当月资源使用量在免费额度内，不收费；如超出免费额度，对超出部分收费。

* 费用主要来自于实时仓库、日志分析两个计费项目，可参考下节[费用优化](/insight/manual/4739/cost-optimization)。

* 当账户进入冻结期时候，会保留30天数据，在此期间系统中存量数据仍会产生计费，如果不需要相应数据请尽快解冻并且删除相关数据

 
# **1.实时仓库（消息队列）计费标准**


|计费项|计费项说明|价格|免费额度（每月）|
|:--:|:--:|:--:|:--:|:--:|:--:|
|实时仓库存储量|新增到实时仓库的数据量| 0.12元/GB|1GB/月|
|实时仓库数量|实时仓库用于存储新增数据| 0.9元/个/天|1个/月</p>免费租赁 1 个仓库/月|

例：
用户共有 5 个实时仓库，扣减免费额度后，一个月(30 天)的实时仓库租赁计费为: 
```0.9【元/个/天】*（ 5【个】-1【个】）* 30【天】 = 108 元```

每月新增加到实时仓库的总数据为 200GB，扣减1GB/月免费额度后，每月的实时仓库存储量计费为:
```0.12【元/GB】*（200【GB】-1【GB】） = 23.88 元```

综上，当月的计费账单为：

```108【元】 + 23.88【元】 = 131.88【元】```

**说明**：为了避免流量突增或者预期之外的数据量进入导致您的账户产生预期外的消费，我们会对新建仓库设置每天 200 GB 数据量的上限。如果您对数据量有更大的需求可以联系销售或者客服。

# **2.日志仓库计费标准**


|计费项|计费项说明|价格|免费额度（每月）|
|:--:|:--:|:--:|:--:|:--:|
|日志仓库存储量|用于存储日志数据，存储时长可自行调节| 0.25元/GB/天|1GB/月 </p>一个月内支持1GB新增数据免费存储31天|

例：
假设每日存储量为 20GB，扣减免费额度后，1个月（30天）的账单金额为：
```0.25【元/GB/天】*（20 【GB】*30【天】- 31【GB*天】）= 142.25元```


# **3.数据立方计费标准**

|计费项|计费项说明|价格|免费额度（每月）|
|:--:|:--:|:--:|:--:|:--:|
|数据立方存储量|用于存储实时日志预聚合指标结果，存储时长可自行调节| 0.99元/GB/天|1GB/月</p>一个月内支持1GB新增数据免费存储31天|
|数据立方租赁数量|用于存储实时日志预聚合指标结果的数据立方数量| 0.49元/个/天|1 个/月</p>免费租赁 1 个仓库/月|

例：用户共创建了 5 个数据立方，一个月(30 天)的数据立方租赁计费为: 

```5【个】 * 0.49【元/个/天】 * 30【天】 = 73.5 元```

每日日增数据为 5GB，每条日志存储 7 天，1个月（30天）的账单金额为：

```0.99【元/GB】* 5【GB】* 7【天】 * 30【天】 = 1039.5 元```

综上，当月数据立方的计费账单为：

```1039.5【元】 + 73.5【元】 = 1113【元】```

# **4.数据推送（API 调用）计费标准**

|计费项|计费项说明|价格|免费额度（每月）|
|:--:|:--:|:--:|:--:|
|API 调用|数据推送 API 调用次数| 2元/百万次|一个月内 1 百万次免费调用|

例:
某月调用 API 次数为 200 万次，扣减免费额度后，一个月的账单金额为:
```(2【百万次】-1【百万次】）*2【元/百万次】 = 2 元```

`注：`此条计费标准用于限制资源滥用，费用非常低。

|:--|:--|
