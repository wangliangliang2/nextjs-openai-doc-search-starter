* 要进行Transform变化的键(`key`): 必填，指定日志中时间所在的字段，时间格式需要为"2006-01-02 15:04:05"，可以使用"date"转换器进行转换。

* 新的字段名(`new`): 必填，指定连接生成的字段名称。

* 时间字段名(`time`): 必填，需要对比的时间字段名（使用该时间减去key对应的时间），时间格式需要为"2006-01-02 15:04:05"，可以使用"date"转换器进行转换。如果不填则需要勾选"确认并添加收集器"高级选项中的"记录读取时间"（具体步骤见下文）,会以日志读取时间来作为对比时间。

注意：若想得到日志读取时间和key的时间差，在确认提交收集器前，勾选高级选项中的“记录读取时间(lst)”。

```
{
		"type":"difftime",
		"key":"created",
		"new":"difftime",
		"time":"read",
}
```

### **示例**：

假设原始数据为:

```
{
		"created": `2020-07-27 18:00:00`,
		"read": `2020-07-27 18:01:00`
}
```

经过 concat transformer 变换后的数据将变为:

```
{
		"created": `2020-07-27 18:00:00`,
		"read": `2020-07-27 18:01:00`
		"difftime": "1m0s"
}
```
