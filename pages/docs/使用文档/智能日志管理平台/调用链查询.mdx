调用链查询用于查看系统中的服务调用链路状态，可能是慢服务、出错的服务。

1.登录智能日志管理平台，搜索 `tracingspan` 仓库和 `tracingservice` 仓库（在 logkit-pro 发送数据到智能日志平台生成的两个仓库，`tracingspan` 仓库里记录 tracing 数据， `tracingservice` 仓库里记录服务数据）中是否有数据：在左侧导航栏中选择**日志仓库-->搜索分析**，搜索 tracing 数据和服务数据。

![](https://pandora-kibana.qiniu.com/tracingspan.png)

2.在左侧导航栏中选择**应用平台-->全链路追踪**，点击`进入应用`。

3.设置查询条件：

* 查询参数说明：

   - 服务：单击下拉框选择服务，支持输入关键字搜索
   
   - 操作：选择当前服务下的操作，如选择 通过 A 服务调用 B 服务的操作
   
   - 标签：通过标签来筛选目标链路，如通过筛选 HTTP 响应码、请求的耗时等标签查询服务中出现异常的链路
   
   - 时间范围：选择链路发生的时间范围。
   
   - 持续时间：设置链路的持续时间，常用于查询慢服务链路。

![](https://pandora-kibana.qiniu.com/tracing/tag.png)

4.链路查询结果提供气泡图展示：气泡个数代表链路条数，气泡大小反应链路持续时间。

5.根据链路查询结果，单击查看链路调用详情。

调用链路详情的具体说明，请阅读下一节[调用链详情](/insight/manual/5118/the-invocation-chain-details)。
