# 推送开发指南（PUSH）
<h3 id="推送-sdk-集成说明"><a href="#%E6%8E%A8%E9%80%81-sdk-%E9%9B%86%E6%88%90%E8%AF%B4%E6%98%8E" ariaHidden="true" tabIndex="-1"><span class="icon,icon-link"></span></a>推送 SDK 集成说明</h3>
<p>七牛IM推送是基于七牛IM技术基础上研发，只需要一次集成，就可以同时拥有推送和IM两大服务，提高研发效率的同时，也会极大降低企业的 IT 支出。</p>
<p>使用七牛IM推送没有额外费用。</p>
<p>由于默认即支持各主流厂商通道，为了进一步减低集成难度，七牛IM推送实现时也内置了证书设置与更新机制。简单来说，就是开发者集成七牛IM推送之后，只需要在控制台设置好各厂商推送的证书，前端将厂商推送 SDK 打包，即可自动完成系统厂商的适配。不再需要针对性调整各种推送令牌的申请和设置。</p>
<p>注意：与其他推送厂商不同的是，七牛IM推送 SDK 专注于推送通道的建设和服务，并不会收集终端信息，如果你有类似广告业务，需要单独集成广告 SDK，或者将业务数据标签通过接口设置后才能使用。</p>
<p>如前所述，由于七牛IM推送 SDK 与 IM SDK 是同一个 SDK，推送功能只是在原有 IM SDK 基础上增加了推送接口。因此集成方式均与 IM SDK 相同，快速集成文档亦可参见七牛IM安卓端快速集成，七牛IM iOS 端快速集成。</p>
<h3 id="sdk-架构"><a href="#sdk-%E6%9E%B6%E6%9E%84" ariaHidden="true" tabIndex="-1"><span class="icon,icon-link"></span></a>SDK 架构</h3>
<p>推送功主要涉及以下三个类：</p>
<pre><code>BMXClient
<span></span>|<span class="hljs-comment">----BMXPushService</span>
<span></span>    |<span class="hljs-comment">----BMXPushServiceListener</span>
<span></span>    |<span class="hljs-comment">----BMXPushManager</span>
<span></span></code></pre>
<p>其中 BMXPushService、BMXPushManager 均为推送设置类，前者是同步调用类，后者异步调用类，实现时根据需要在两者中二选一即可。</p>
<p>其他功能类分别是：</p>
<ul>
<li>BMXCallBack : 无类型接口回调</li>
<li>BMXDataCallBack : 泛型类型带数据回调</li>
<li>BMXPushServiceListener : 推送事件监听</li>
<li>BMXMessage : 推送消息</li>
<li>BMXUserProfile : 推送用户信息</li>
</ul>
<p>下文以安卓 SDK 为例介绍推送 API。</p>

<h3 id="厂商推送集成"><a href="#%E5%8E%82%E5%95%86%E6%8E%A8%E9%80%81%E9%9B%86%E6%88%90" ariaHidden="true" tabIndex="-1"><span class="icon,icon-link"></span></a>厂商推送集成</h3>
