# 计费组成

智能多媒体服务费用的各项组成，包括：

1. 通过 **调用多媒体 api 接口**，产生的费用。

2. 在智能多媒体控制台，通过 **工作流** 发起的任务处理，产生的费用。

 - 工作流可以编排多任务处理流程，会对处理成功的节点进行收费，对处理失败的节点不会进行收费。

 - 由系统故障导致任务处理失败，都不进行收费。

   > 例如：一个任务有几个处理节点，其中2个节点处理成功，其他节点是系统故障导致的处理失败，则所有节点都不会进行收费。

> - 转码费用计算公式：转码费用 = 输出文件时长 x 转码单价 
> - 输出时长：对于每月转码输出文件，按分钟计费，累计总时长不足一分钟不计费
> - 价格单位：元／分钟。
> - 输出规格按输出视频分辨率的长边和短边属于输出规格划定的范围进行判定，方式如下：
  以输出SD (1080 x 720) 规格为例，输出视频的分辨率长边不大于1080且短边不大于720则属于该输出规格。
  如输出视频的长边大于1080或输出视频的短边大于720，该输出视频属于更高输出规格。
> - 使用 avthumb 命令时，当 `VideoCodec` 和` AudioCodec` 两个参数值同时设为 copy，即不进行音视频编解码操作，以音视频转封装计费。
> - 使用 avthumb 命令时，且同时启用锐智转码，即使用 `avsmart `参数，按照锐智转码价格计费，与普通音视频转码计费有区别。
> - 音视频分段（segment）的计费，同音视频转封装。

# 计费示例
**示例1**

某公司2019年1月使用七牛转码服务，采用H.264编码方式，普通转码转出1080 x 720 分辨率视频时长100分钟。使用转封装1000分钟，则产生的费用如下：
```
转码费用=100（分钟）* 0.0189（元/分钟）=1.89（元）
装封装费用=1000（分钟）* 0.003（元/分钟）=3（元）
```

**示例2**

某公司2018年5月使用七牛图普图片鉴黄服务，共发起500万次鉴黄请求，其中结果确定的次数为480万次，结果不确定的次数为20万次，则当月使七牛鉴黄服务产生的费用为：
```
确定的结果费用：0.25(元/百次) * 300(万次) + 0.23(元/百次) * (480万次 - 300万次) = 11640(元)
不确定的结果费用：0.0625(元/百次) * 20(万次) = 125(元)
```
