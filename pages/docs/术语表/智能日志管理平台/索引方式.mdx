智能日志平台支持对字符串类型的数据进行倒排索引，可以使用搜索引擎的技术对于日志进行索引，并且支持各种灵活的搜索方式。其中索引的方式包括两种：

**分字段索引**

分字段索引可以允许用户挑选字符串字段，选择灵活的分词和索引方式。

![](https://pandora-kibana.qiniu.com/field_index.png)

可以参考 [Lucene 语法](/insight/manual/4871/search-syntax)（如 log:Error 代表在 log 字段中搜索 Error 单词）
对数据进行搜索、过滤和分析。使用分字段索引的时候可以指定仅仅对某些字段进行分词，降低存储成本，提高写入和检索效率。

**全文索引**

1.为仓库开启全文索引表示将仓库所有数据看作一个字段，并为其指定固定的分词索引方式（标准分词器/空白分词器/中文分词器）。

![](https://pandora-kibana.qiniu.com/context_index.png)

对于全文索引仓库，您在检索的时候直接输入文本也可搜索，如直接在搜索框中输入：

```
"writepoints"
```

2.开启了全文索引的仓库中的字段只能使用不分词索引方式。

![](https://pandora-kibana.qiniu.com/bufenci.png)

这就意味着如果您需要搜索指定字段的具体内容，仍然需要输入您想搜索的完整内容。如：

```
operation:"writepoints"
```

如果您的字段特别多，或者字段是非常复杂的嵌套结构，此时我们非常推荐您选择全文索引，会使搜索更加便利和灵活。
不过要注意：全文索引会增加存储成本，而且无法在存储时挑选部分字段索引，也无法单独为某些字段设置分词器。
 
![](https://pandora-kibana.qiniu.com/logdb/%E5%85%A8%E6%96%87%E7%B4%A2%E5%BC%95.png)
