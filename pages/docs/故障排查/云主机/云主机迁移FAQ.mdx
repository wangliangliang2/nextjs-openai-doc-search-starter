# QSMC是否支持断点续传？
支持。数据传输中断后，重新运行客户端并重新启动迁移任务即可继续迁移。

# 迁移中断或提示失败时，如何处理？
迁移任务中断或失败时，处理方式如下：

- 当 QSMC 客户端程序异常退出或者迁移进度卡顿时，可以尝试重新运行SMC客户端并重启迁移任务以恢复迁移。
- 如果迁移任务状态为出错，您可以在 QSMC 控制台查看迁移任务的日志文件，定位错误原因。
- 如还有问题，请提交工单。

# 中转实例的规格有哪些？
QSMC会按下列顺序，根据库存情况，依次选择符合条件的突发型t5、突发型t6 实例规格来创建中转实例。
- 1 vCPU 2 GiB
- 1 vCPU 4 GiB
- 2 vCPU 2 GiB
- 2 vCPU 4 GiB
- 2 vCPU 8 GiB
如果以上实例规格均库存不足时，系统会选择其他性能和价格兼顾的实例规格。

# 关于中转实例，我需要注意什么？
有关中转实例的注意事项如下：
- QSMC自动创建、启动、停止和释放中转实例No_Delete_SMC_Transition_Instance。为保证顺利完成迁移任务，请勿干预中转实例的运行状态。
- 中转实例的默认安全组在入方向开放了8080和8703端口，这是中转实例的迁移服务端口，请勿修改或删除该安全组配置。
- 迁移任务完成后，中转实例会被自动释放，如果迁移失败，需要手动释放实例。

# 误释放了中转实例怎么办？
如果误清理了中转资源，您可以删除当前的迁移任务，重新为迁移源新建并启动迁移任务。


# 我的待迁移服务器需要在出方向访问哪些公网地址和端口？
确认源服务器能访问以下服务地址和端口。
- 服务器迁移中心SMC：https://smc.aliyuncs.com443端口。
- 中转实例：公网IP地址8080和8703端口。当您使用内网传输的网络模式时，需访问私有IP地址。

- 如何安装Rsync？
请您根据源服务器的操作系统选择相应的命令安装Rsync。
- CentOS：运行yum -y install rsync。
- Ubuntu：运行apt-get -y install rsync。
- Debian：运行apt-get -y install rsync。
- SUSE：运行zypper install rsync。
其他发行平台系统：参见发行版官网的安装文档。

# 我想要重新导入迁移源，怎么办？
您需要先删除迁移源后，再重新运行客户端导入迁移源。若迁移源已和迁移任务关联，请先删除与之关联的迁移任务，再删除迁移源。


# 迁移任务多久过期？过期后会怎样？
通过 QSMC 控制台创建迁移任务时，由于控制台不提供过期时间的设置方式，因此任务的默认有效期为30天
