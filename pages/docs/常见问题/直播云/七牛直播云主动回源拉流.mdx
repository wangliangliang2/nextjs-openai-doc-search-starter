# 简述
一些特定的业务需求，自己搭建了流媒体服务器，使用 rtmp 推流到自己的节点服务器，但是受限于播放并发量和延迟，服务器负载不了的情况下，可以配置七牛直播云主动回源拉流，然后通过七牛的直播云节点进行分发。
</br>
# 优势
1.减小源站服务器负载压力  
2.缩短直播延迟，提高实时性  
3.直播落存储到七牛，节省源站存储空间，备份数据，有转码需求七牛也可以提供
</br>
# 支持源站的视频封装格式
| 格式 | 支持 | 示例格式 |
| -- | :--: | :--: |
| RTMP | V | `rtmp://a.com/live/test` |
| FLV | V | `http://a.com/live/test.flv` |
| HLS | X | X |
</br>
# 如何提交配置
请提交 [工单](https://support.qiniu.com/tickets/category) ，说明如下配置详情：  
1.**提供源站的播放地址**  
2.**七牛账号**，**七牛的直播空间名称**，以及需要配置的**播放域名**
</br>

# 工作流程
假如配置到七牛 `b.com` 播放域名上，源站地址为 `rtmp://a.com/live/test` 则回源拉流配置后，使用七牛的播放地址为  
`rtmp://b.com/<直播空间名>/test`  
`http://b.com/<直播空间名>/test.m3u8`  
`http://b.com/<直播空间名>/test.flv`
假如请求了三个地址中任意一个，会直接触发七牛直播云进行向 `rtmp://a.com/live/test` 的回源拉流请求。
