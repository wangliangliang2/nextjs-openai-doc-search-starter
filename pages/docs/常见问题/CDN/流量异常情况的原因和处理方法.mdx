|Q1:流量突增、流量明显高于正常业务量，应该如何处理？|
| ---- |
|1.可以 [下载日志](https://portal.qiniu.com/domain/clog) 进行 [统计分析](https://developer.qiniu.com/kodo/kb/1451/how-to-own-space-ip-access-number)。<br>2.可以针对CDN域名，在 [告警配置](/fusion/kb/4059) 开启配置 告警，当使用量超过您设置的告警值会进行通知。<br>3.如果通过分析，得出流量确实产生于被盗刷或者是被攻击，可以在域名 “防盗链配置” 中，配置防盗链。 |

<br>
 
|Q2:域名被攻击，为何停用或者删除了域名后，依然还有流量？|
| ---- |
|当域名仍然解析到七牛，七牛对于攻击的请求会给出非 200 的响应，这种响应有通信头的流量的消耗，当攻击频次巨大时，会产生计费的流量。<br>当域名是自定义域名，建议调整解析到 1.1.1.1。若调整解析为空，攻击者可能更换攻击方式，绑定 host 攻击。<br>当域名是七牛提供的域名，需要提交工单联系七牛调整域名解析。 |

<br>

|Q3:为什么使用http下载消耗的却是https流量？|
| ---- |
|域名如果开通了 https，那么统一按照 https 收费。<br>如果有些请求使用 http，可以新增一个专门用来 http 访问的域名。|

<br>

|Q4:为什么资源删除后依然会产生流量？|
| ---- |
|这个是资源在访问后，会在cdn中生成缓存，下次访问的时候直接会从cdn中加载资源，而不用回源到源站去加载资源，有效缩短加载时间，这个本身就是cdn的优势所在。所以文件删除后需要调用刷新接口去删除cdn的缓存，这样才能保证文件删除后无法继续访问。如何刷新缓存：1. 通过api 接口刷新：https://developer.qiniu.com/fusion/api/1229/cache-refresh 2. 通过portal.qiniu.com 控制台刷新缓存，点击左侧cdn => 刷新预取|
